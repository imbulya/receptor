# Receptor (Рецептор)

Учебный проект для отработки курса «Разработка мобильных приложений».

Android‑приложение для управления домашними запасами: продукты по категориям, сроки годности, подбор рецептов и список покупок.

## Возможности

- **Мои продукты**: категории, добавление/удаление продуктов и единиц с датой годности.
- **Уведомления о сроках**: напоминания о просроченных и “скоро истекающих” продуктах (WorkManager).
- **Рецепты**: локальная база рецептов (`assets/recipes.json`) + пользовательские рецепты.
- **Список покупок**: добавление вручную и перенос купленного в “Мои продукты”.
- **Оффлайн‑режим**: данные и рецепты хранятся на устройстве.

## Технологии

- Java
- AndroidX + Material
- WorkManager
- Glide (загрузка изображений по `imageUrl`, включая `file:///android_asset/...`)

## Сборка и запуск

### Вариант 1 — Android Studio

1. Откройте проект в **Android Studio**.
2. Дождитесь синхронизации Gradle.
3. Соберите и запустите на устройстве/эмуляторе.

CLI (опционально):

```bash
./gradlew :app:assembleDebug
```

### Вариант 2 — скачать APK с GitHub

Если APK опубликован в **Releases** (или как **Artifacts** в GitHub Actions), его можно скачать на телефон и установить:

1. Откройте страницу **Releases** репозитория на GitHub.
2. Скачайте `app-debug.apk` (или `app-release.apk`) на телефон.
3. Установите APK (при необходимости разрешите установку из неизвестных источников).

## Структура данных рецептов

Файл: `app/src/main/assets/recipes.json`

Минимально используемые поля:

- `id` (string)
- `title` (string)
- `imageUrl` (string, опционально)
- `cuisine` (string)
- `type` (string)
- `timeMinutes` (number)
- `ingredients` (array: `{ name, amount }`)
- `steps` (array of string)
